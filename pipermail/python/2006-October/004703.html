cavte... snazim sa spravit komunikaciu cez pipu. Vytvorim ju cez
os.pipe(). Na jednej strane (read) potom spravim os.fdopen, aby som
dostal file object. Na druhej strane (write) pracujem s file
descriptorom. Ked vsak skoncim zapisovat do pipy a zatvorim ju, ta
strana, ktora cita zostane vysiet na f.read(1), f.readline() atd. Ako mu
poslem eof, nech si to mozem osetrit?
