Ja vyvijam stale vo verzii 2.5 a po verzii 3.0 zatial vobec netuzim. Zostanem pri verzii 2.5 este zrejme este dost dlho.
V mojom pripade som pri verzii 2.5, pretoze python uz dlho pouzivam,  mam napisanych kopu skriptov, ktore mi bez problemov bezia na dvoch OS, ktore pouzivam a nechcem ich teraz prerabat na verziu python 3.0.
 
Predpokladam, ze chces pouzivat py2exe. 
Problem je, ze prave teraz ma python prechodne obdobie - takze ak teraz zvolis python 3.x nebudes mat py2exe este hned teraz k dispozicii.
Keby som teraz s pythonom zacinal ja a  chcel pouzivat py2exe, tak by som asi zvolil ako kompromis python 2.6. 
Je k nemu na stiahnutie binarka py2exe-0.6.9.win32-py2.6.exe.
 
Na druhej strane ak sa teraz naucis python 2.x a napises v nom nejake skripty, budes ich mozno raz musiet prerabat na verziu 3.x. 
A to len kvoli tomu, ze si chcel mat hned teraz k dispozicii py2exe.
Takze 2. moznost je vykaslat sa teraz na py2exe a zacat pouzivat python 3.x
 
3. moznost, je dat si este na urcity cas s pythonom pauzu a naucit sa iny skriptovaci jazyk (Perl, Groovy, Ruby?...)
