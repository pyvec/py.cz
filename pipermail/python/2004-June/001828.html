V interview se casto pletou pojmy strong typing se static typing. C a C++
jsou weak typing + static typing jazyky.

Zajimalo by mne, proc bych mel podle gvr hotovy a fungujici program v pythonu
prepisovat do C++. Co bych tim krome moznych problemu ziskal?  Muj cil prece
neni vytvorit C++ program, ale vyresit urcity problem.

Ono je to spis tim, ze aplikace v C++ maji nalepku profesionality, zatimco
python scriptovaciho prototypu. Casto slysim: ten program co jste napsal
je dobry, jeste to prepiste do C++ a pak ho koupime. 

Spravne reseni tohoto problemu neni prepisovat program, ale python vice
integrovat a zamaskovat, takze neni na prvni pohled tak napadny. Tak
to ma napriklad clamav pro windows (py+wxwin).
